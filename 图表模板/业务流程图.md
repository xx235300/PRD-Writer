# 业务流程图 Mermaid 模板

> 本模板用于生成 Mermaid 业务流程图（flowchart）。

---

## 基础语法

```mermaid
flowchart TD
  A([开始]) --> B{判断条件}
  B -->|是| C[处理1]
  B -->|否| D[处理2]
  C --> E([结束])
  D --> E
```

---

## 节点形状

| 形状 | 语法 | 用途 |
|------|------|------|
| 圆角矩形 | `A([文字])` | 开始/结束节点 |
| 矩形 | `A[文字]` | 普通处理节点 |
| 菱形 | `A{文字}` | 判断/决策节点 |
| 圆形 | `A((文字))` | 连接点/辅助 |
| 六边形 | `A{{文字}}` | 准备/状态 |

---

## 连接线语法

| 语法 | 说明 |
|------|------|
| `A --> B` | 实线箭头 |
| `A --- B` | 实线无箭头 |
| `A -->|文字| B` | 带标签箭头 |
| `A -.-> B` | 虚线箭头 |
| `A ==> B` | 加粗箭头 |

---

## 示例：用户反馈处理流程

```mermaid
flowchart TD
  Start([用户提交反馈]) --> Input{输入验证}
  Input -->|通过| Classify{自动分类}
  Input -->|失败| Reject[提示重新填写]
  Reject --> End1([结束])
  
  Classify -->|置信度高| Assign[自动分配客服]
  Classify -->|置信度低| Manual[人工分类]
  
  Assign --> Notify[通知客服]
  Manual --> Assign
  
  Notify --> Process{客服处理}
  Process -->|需要用户确认| Wait[等待用户确认]
  Process -->|无需确认| Close[关闭工单]
  
  Wait --> Confirm{用户确认}
  Confirm -->|满意| Close
  Confirm -->|不满意| Reopen[重新打开]
  Reopen --> Process
  
  Close --> NotifyUser[通知用户]
  NotifyUser --> End2([结束])
```

---

## 方向语法

| 方向 | 语法 | 说明 |
|------|------|------|
| TD / TB | `flowchart TD` | 从上到下（默认） |
| BT | `flowchart BT` | 从下到上 |
| LR | `flowchart LR` | 从左到右 |
| RL | `flowchart RL` | 从右到左 |

---

## 设计规范

### 节点命名

- 使用有意义的名称，如 `SubmitForm` 而不是 `A`
- 开始节点用 `Start` 或 `A`
- 结束节点用 `End` 或 `Z`
- 判断节点用 `{ }` 包裹

### 连接线标签

- 在判断分支上添加 `|是|`、`|否|` 等标签
- 标签帮助理解流程逻辑

## 快速生成模板

### 简单线性流程

```mermaid
flowchart TD
  A([开始]) --> B[处理步骤1]
  B --> C[处理步骤2]
  C --> D[处理步骤3]
  D --> E([结束])
```

### 带判断流程

```mermaid
flowchart TD
  A([开始]) --> B{判断条件}
  B -->|是| C[处理A]
  B -->|否| D[处理B]
  C --> E([结束])
  D --> E
```

### 完整业务流程模板

```mermaid
flowchart TD
  Start([开始]) --> Input[接收输入]
  Input --> Validate{验证输入}
  Validate -->|通过| Process[业务处理]
  Validate -->|失败| Error[错误处理]
  Process --> Decide{处理结果}
  Decide -->|成功| Success[返回成功]
  Decide -->|失败| Fail[返回失败]
  Success --> End1([结束])
  Fail --> End2([结束])
  Error --> End3([结束])
```
